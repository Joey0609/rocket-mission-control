# Copilot 工作区指令

## 目录职责

- `server/httpServer.js` 是唯一的状态入口，负责读取 JSON、归一化模型、生成快照和处理 API。
- `static/js/admin.js` 负责管理端总控、当前型号、发射时间、HOLD、遥测按钮和本地 tick 驱动。
- `static/js/visitor.js` 负责访客页展示、时间轴、上一事件 / 下一事件、遥测面板和本地 tick 驱动。
- `static/js/admin-preview.js` 只负责编辑器预览和草稿同步，不要在这里放业务状态推进逻辑。
- `static/js/admin-editors/*` 放各类编辑器的表单与校验公共逻辑，尽量保持纯函数。

## 状态规则

- 遥测、倒计时、事件节点这些状态尽量通过“当前时间 + 配置”派生，不要为了展示再单独起一条定时广播链。
- 页面已有的 tick 逻辑要优先复用，新的时间驱动逻辑应挂到现成 tick 上。
- 服务器只负责生成快照和接收显式操作，不要把 UI 派生状态做成独立的持久真值。

## 配置规则

- 事件观察点统一写在 `events[].observation`，不要再新增独立 `observation_points` 主数据。
- 仪表盘节点的遥测自动开关统一写在 `dashboard_editor.nodes[].telemetry_auto`。
- 新增字段时，后端归一化、前端草稿、保存和回读都要同时处理。

## 兼容规则

- 读取旧数据时允许兼容旧字段，但保存时只输出当前结构。
- 迁移字段时要检查调用链：API 返回、SSE 快照、编辑器草稿、预览渲染、访客页展示都要一起看。

## 约束规则

- 本仓库仅允许 Node.js Web 服务架构（前后端同服）。
- 不要再引入 Python、Flask、bat、vbs 启动链。
- 页面主计时必须只有一个，0 秒后自动切换正计时，不允许同时显示两套主时钟。
- SSE 优先；只有在明确需要时才补兜底轮询。

## 代码整理原则

- 先找调用链，再拆分函数，不要先写新定时器再补逻辑。
- 优先抽取纯函数和共享规则，减少同一规则在多个文件里各写一遍。
- 改动前先看全局状态和隐式依赖，特别是 `lastState`、`missionAnchor`、`dashboard_editor`、`telemetry_enabled`。
- 任何改动都要预判边界：空型号、空事件、缺字段、时间没传、节点未排序、状态不同步。
